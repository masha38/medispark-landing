# 메디스파크 랜딩페이지 Apps Script 명세서

## 목적

- 랜딩페이지에서 수집한 고객 DB를 구글시트에 자동 저장
- 저장 항목에 `이름`, `연락처`, `개발자정보`를 반드시 포함
- 운영 분석용으로 `관심타입`, `상담유형`까지 함께 저장
- 저장 성공 후 사용자 전화 연결 유도

## 운영 정보

- 대표번호: `010-4896-1691`
- 개발 담당자: `장문희`
- 페이지명: `메디스파크 랜딩페이지`

## 저장 흐름

1. 사용자가 랜딩페이지에서 `상담하기` 클릭
2. 이름과 연락처 입력
3. 프론트엔드가 Apps Script Web App URL로 POST 요청
4. Apps Script가 구글시트에 한 줄 저장
5. 저장 성공 응답을 받으면 `tel:010-4896-1691` 실행

## 필수 저장 항목

- `submittedAt`
- `name`
- `phone`
- `developerName`
- `interestType`
- `consultType`

## 권장 저장 항목

- `pageName`
- `leadType`
- `inflowSource`
- `deviceType`
- `userAgent`

## 숨김값 기본 설정

- `developerName = 장문희`
- `pageName = 메디스파크 랜딩페이지`
- `leadType = 상담하기`
- `interestType = 사용자가 선택한 관심 평형`
- `consultType = 전화상담 / 방문예약 / 문자안내`

## Apps Script 배포 방식

- 구글 스프레드시트 생성
- `확장 프로그램 > Apps Script` 진입
- `apps_script.gs` 코드 붙여넣기
- `SPREADSHEET_ID`, `SHEET_NAME` 수정
- `배포 > 새 배포`
- 유형: `웹 앱`
- 실행 사용자: `나`
- 액세스 권한: `모든 사용자`
- 배포 후 Web App URL을 HTML의 `scriptUrl`에 반영

## 예외 처리 권장사항

- 저장 실패 시 사용자에게 `잠시 후 다시 시도해주세요` 안내
- 전화 연결보다 저장이 먼저 성공하도록 처리
- 저장은 성공했지만 전화 앱 실행이 실패해도 DB는 남도록 구성
- 시트 헤더 순서와 Apps Script의 `appendRow` 순서는 반드시 동일하게 유지
